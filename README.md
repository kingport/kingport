## ✨ 我做过的几件比较有代表性的事

### 01. 从 0 到 1 起草并落地公司内部 UI 组件库

我参与并推动了公司内部通用 UI 组件库的建设。
这件事并不只是封装按钮、弹窗和表单，而是围绕公司品牌设计、后台系统场景和团队协作方式，逐步整理出一套可以在多个内部系统中复用的前端基础能力。

它带来的价值是：

- 内部系统的视觉与交互开始变得统一
- 页面开发减少了大量重复造轮子
- 新需求接入成本更低，老系统的维护也更稳定

我比较看重的不是“做了一个组件库”，而是它让开发方式从“页面堆砌”慢慢变成了“能力沉淀”。

---

### 02. 推动试算能力从“分散开发”变成“统一 SDK”

之前不同业务线都在各自实现试算逻辑，看起来是多套模块，实际上底层内容高度重复。

我参与把这部分能力重新抽象，统一整理成试算组件 SDK，让多个业务线能够以更一致的方式接入。

这件事的意义在于：

- 降低重复开发
- 减少逻辑分散带来的维护成本
- 让试算能力更接近“平台能力”，而不是“项目私有实现”

对我来说，这不只是“写了一个 SDK”，而是做了一次真正的平台化收敛。

---

### 03. 在核心交易系统里持续做“复杂问题简单化”

核心交易系统最有挑战的地方，很多时候不在技术本身，而在于：

- 业务规则复杂
- 系统协同多
- 页面链路长
- 需求变化快

我长期在做的一件事，就是尽量把这些复杂度压到更合理的范围：
把结构拆清楚、把模块边界划清楚、把通用逻辑收敛出来，让系统更容易继续演进，而不是越做越重。
